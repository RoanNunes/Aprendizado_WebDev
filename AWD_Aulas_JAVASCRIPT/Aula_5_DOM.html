The DOM

Aqui sobre o DOM e exclusivamente ele, pois ele é lirealmente todo arquivo criado pelo navegador e aquele que o Javascript tem capacidade de comunicar-se e manipular.
O DOM é uma sigla de "Document Object Model", seu semelhante em português seria "Modelo de Documento por Objetos".
O DOM é a camada interface entre a linguagem de programação e o HTML/CSS neste contexto.
O DOM não é algo exclusivo da linguagem Javascript, apenas é um metodo geralmente usado em desenvolvimento Web.

O navegador interpreta a página como um formato tipo árvore, a qual não é necessáriamente binária.
Uma interpretação interessante do que seria o DOM seria a seguinte: DOM é a linguagem comum entre o HTML/CSS e Javascript e como eles se comunicam entre si.

A estrutura do DOM

Como dito anteriormente, o DOM é uma interface entre um documento e uma (ou mais) linguagem de programação.
Ele pode ser seguimentado em 3 (três) partes:

  • CORE DOM - É o modelo padrão de qualquer tipo de documento (que usa esse modelo);
  • XML DOM - É o modelo padrão de todo documento XML;
  • HTML DOM - É o modelo padrão de todo documento HTML

HTML DOM

HTML DOM é principal modelo a ser analizado aqui.
Ele é a estrutura de objeto e camada de interface necessária para programação.
O HTML DOM define todo elemento HTML (tags) como um objeto.
A nomeclatura de um HTML DOM é a seguinte.

  • Todo estado HTML é um evento (event).
  • Todo acesso HTML é um método (method).
    • Todo método no HTML DOM é uma ação a qual pode ser executada em um elemento HTML.
  • Todo atributo HTML é uma propriedade (properties).
    • Toda propriedade no HTML DOM é um valor o qual pode ser atribuido o modificado em um elemento HTML.

A leitura do HTML DOM

Em um documento HTML DOM há centenas de objetos, além daquele que são elementos HTML.
Existe uma lista com todos os comandos possíves de executar e capturar a informação deles.
A lista completa se encontra aqui: Lista completa por W3Schools
Segue destacado algum desses objetos:

  • document.getElementByTagName("identificador");
    • Busca e captura todo elemento com tag HTML solicitada.
    Por ele capturar todo elemento com tal tag, ele retorna uma coleção. É semelhante a um vetor/matriz, no entanto mais limitado.
  • document.getElementByClassName("identificador");
    • Busca e captura todo elemento com a classe solicitada.
    Por ele capturar todo elemento com tal classe, ele retorna uma coleção. É semelhante a um vetor/matriz, no entanto mais limitado.
  • document.getElementById("identificador");
    • Busca e captura um único elemento com a id solicitada.
  • document.querySelectorAll("identificador");
    • Busca e captura todo elemento com o seletor CSS solicitada.
    Incluí todo tipo, como: id, classes, atributo, psudo-class, etc
    Por ele capturar todo elemento com tal classe, ele retorna uma coleção. É semelhante a um vetor/matriz, no entanto mais limitado.
    [ANOTAÇÃO] Sim, isso é um método que também pode selecionar classes, mas não está limitado a somente isso.

Lembrando que isso já são os comandos em Javascript! E podem ser manipulados como qualquer variável, matriz ou método dado o contexto.

A modificação do HTML DOM

Por o DOM ser estruturado como um árvore, o método de modificação direta nela será através dos nodos.
É sempre necessário a criação do nodo primeiro, depois alocar ele na posição desejada.
Segue as funções mais comuns de manipulação do HTML DOM:

  • document.createElement("HTML Tag")
    • Cria um elemento HTML.
    [ANOTAÇÃO] Ele cria o inicio e fim de um elemento HTML (tag de inicio e fim)
  • document.appendChild(element)
    • Acrescenta um elemento HTML.
    [ANOTAÇÃO] Existe métodos para especificar onde o elemento deve ser acrescentado.
  • document.removeChild(element)
    • Remove um elemento HTML.
    [ANOTAÇÃO] Existe métodos para especificar qual elemento deve ser removido.
  • document.replaceChild(element new, element old)
    • Substituí um elemento HTML por outro.
  • document.write("String")
    • Escreve no fluxo HTML próprio.
