Objetos e Eventos

Semelhante a ideia de funções, a de agrupar instruções para reutilizar, objetos agrupam variáveis de contexto semelhante para melhor referênciar em meio ao código.
E eventos são formas do Javascript fazer uma página HTML "reagir" dado um contexto, semelhante a 'pseudo-classes' de CSS.
Embora esses temas serem distintos, são formas de melhor cordenar o Javascript e suas interações com HTML e CSS.

Objetos

Objetos são conjutos de variáveis (ou de dados) homogênios ou heterogênios.
Além de variáveis, objetos podem conter funções próprias as quais só podem ser utilizadas pelo próprio objeto.
Um objeto é declarado da mesma forma que uma variável, no entanto, seus atributos são declarados de forma diferente.
Toda propriedade de um objeto é dado um nome e um valor, definido por (:) dois pontos.
Cada propriedade é separada por (,) vírgula, diferente de uma variável normal que é (;) ponto e vírgula.
É boa prática que toda função dentro de um objeto retorne algum valor, ou seja, ela não seja do tipo void (tipo vazio, sem retorno).
Segue alguns modelos:
let pessoa = {
primeiroNome: "Joey",
segundoNome: "Nostradamos",
idade: 74,
altura: 1.72
}
let carro = {
id: 00001,
marca: "Toyota",
qntPortas: 4,
força: 1.1,
posicao_x: 0,
posicao_y: 0,
acelerar: function (){
this.posicao_x += 1;
return: posicao_x; }
}

Eventos

Eventos são ocorrências que podem ocorrer na página Web. Ele se assemelham ao pseudo-classes de CSS.
Diferente de pseudo-classes, eventos em Javascript estão relacionado ao 'DOM' ou ao documento HTML como um todo, incluindo cada interação e processo ocorrido nele.
Para todo evento é necessário um "event listener", uma verficação que é constatemente feita se o caso foi realizado e tratar de acordo.
Existe uma série de tipos de eventos, como iterações, verificações, carregamento, tratamento de erro e etc.
Formas como um "event listener" pode se demonstrar em HTML:
(button type="button" onclick="imprimirCarro()")Imprimir carro(/button)
(div onmouseover="executaFuncao()")(/div)
(h1 onkeypress="executaFuncao()")(/h1)
Formas como um "event listener" pode se demonstrar em JAVASCRIPT:
button.addEventListener("click", imprimirCarro);
div.addEventListener("onmouseover", executaFuncao);
div.addEventListener("onmouseover", (evt) => executaFuncao(param1, param2, ...));
div.addEventListener("onmouseover", executaFuncao.bind(this, param1, param2, ...));
Perceba que em 'addEventListener' possui um método específico para executar a função, incluindo seu método de escrita.
Uma alternativa para executar uma função com parametros dentro de um EventListener seria div.addEventListener("onmouseover", (evt) => executaFuncao(param1, param2, ...))
Existe uma lista grande de todo evento que é identificado pelo DOM.
Esses eventos podem ser etapas no processo de carregar uma página, iterações possíveis com o mouse, iterações possíveis com o toque e outros.
Segue alguns mais comuns:

  • onchange
    • qualquer modificação de um elemento HTML
  • onclick
    • O mouse do usuário clica um elemento HTML
  • onmouseover
    • O mouse do usuário entra ou 'sobrevoa' de um elemento HTML
  • onmouseout
    • O mouse do usuário sai de um elemento HTML
  • onkeydown
    • O usuário pressiona uma tecla
  • onload
    • O navegador termina de carregar a página.
