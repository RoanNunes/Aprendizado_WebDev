Seletores de CSS

Há 3 (três) seletores definidos, ou próprios e há 2 (dois) pseudo seletores.
Seletores definidos são aqueles o qual a referência é um termo, ou predeterminado pelo HTML ou criado para identificar pelo CSS;

  • Tag(s) (HTML) - Exemplo: body, h1, h2, p, spain, table, li, ul, ol, ...
  • Class(es) - Exemplo: .container, .nav, .caixa, .nome, .contexto, .descrição, ...
  • Id(s) - Exmeplo: #Nome, #Header, #Hotel, #Bola, ....

    • Pseudo seletores são definidos por sua referência ser relativa, sendo determinada por um padrão, algoritmo ou um argumento;
  • 'Pseudo-class' - Exemplo: :nth-child, :hover, :link, :first-child ...
  • 'Pseudo-element' - Exemplo: ::after, ::before, ::first-letter, ::first-line, ::marker, ::selection

    • Seletores definidos

    São eles: tag, class, id.
    Podemos usa-lós individualmente, múltiplos, ou combinados.

  • Individulmente - Exemplo: h1 {}; .caixa {}; #Nav {}; ...
  • Múltiplos - Exemplo: h1, .contexto {}; .descrição, #Bola {}; ...
  • Combinados - Exemplo: body > h4 {}; .contexto + img {}; ...

    • Seletor: TAG

    TAG(s) são os identificadores usados no próprio HTML.
    Caso não haja tratamento, toda estilização feita por TAGs são feita em toda instância pela página ou arquivo .html.
    Segue o seguinte exemplo:

    h5{ font-size: 1.6em; color: white; background-color: black; }
    Eu sou um h5!
    Não importa qual seja.
    Eu sou estilizado!

    Seletor: CLASS

    Class(es) são os identificadores criados pelo desenvolvedor e podem se referenciados múltiplas vezes.
    No CSS, toda classe é determinada pelo nome haver um ponto (.) como préfixo.
    Elementos HTML podem referênciar uma ou múltiplas classes ao mesmo tempo, podendo haver sobreposição entre as propriedades das classes diferentes;
    Genericamente, quanto mais a direita a classe é referênciada, mais prioridade ela possui na sobreposição (como se estivesse empilhando classes);

    .container-flexbox{ display: flex; width: 100%; justify-content: center; margin: 20px auto; }
    .caixa{ width: 200px; height: 200px; }
    .caixa-curvada{ width: 200px; height: 200px; border-radius: 10px; }
    .caixa-bordada{ border-color: black; border-style: solid; border-width: 0.1em; }
    .azul{ background-color: aqua; }
    .vermelho{ background-color: red; }
    Sou uma caixa.
    Sou uma caixa curvada.
    Sou uma caixa, sou azul.
    Sou uma caixa, sou azul..?

    Seletor: ID

    ID(s) são os identificadores criados pelo desenvolvedor e podem se referenciados somente uma única vez.
    No CSS, todo ID é determinado pelo nome haver um ponto (#) como préfixo.
    Elementos HTML podem referênciar uma ou múltiplos IDs ao mesmo tempo, contando que os mesmos não foram usados ainda;
    Genericamente, quanto mais a direita o ID é referênciado, mais prioridade ele possui na sobreposição (como se estivesse empilhando itens);

    #FotoUnica{ width: 200px; height: 200px; border-radius: 50%; background-color: brown; }
    #Header{ width: 400px; height: 70px; border-radius: 10px; border-color: black; border-style: solid; border-width: 0.1em; }
    #Olimpica{ width: 100%; height: 100px; border-color: black; border-style: dotted; border-width: 0.3em; background-color: red; }
    Sou uma "foto"
    Sou um header
    Sou um header?

    Seletores combinados

    Assim como podemos selecionar um ou múltiplos elementos e modifica-los ao mesmo tempo, podemos selecionar através do relacionamento entre eles.
    Um relacionamento de elementos Html é derivado da organização e estruturação da página HTML.
    E para determinar qual tipo de relacionamento desejamos agregar ao escopo do seletor há uma simbologia específica:

  • (espaço) - seletor descendente
  • (>) - seletor filho
  • (+) - seletor de irmão adjacente
  • (~) - seletor geral de irmãos

    Pseudo-Seletores ou Seletores relativos

    São eles: Pseudo-classes, Pseudo-elementos.
    Pseudo-Seletores são seletores relativos, que indicam um estado do objeto, algoritmo, condição, ou argumento.

  • Pseudo-class - Representado pelo préfixo de (:) dois pontos antes do nome;
  • Pseudo-element - Representado pelo préfixo de (::) duas pontuações seguidas de dois pontos antes do nome;

    • Diferentes dos seletores definidos, o desenvolvedor não define o nome, ou o que é, ou o que faz. Existe um conjunto de comando já imbutinos no CSS3.

    Seletor: Pseudo-Class

    Pseudo-Classes usualmente definem um estado ou situação o qual um elemento pode se encontrar.
    Aqui serão apenas explicados alguns por seu potencial, mas a lista completa pode ser encontrada aqui: Pseudo-Classes por W3shools
    Para o seguinte exemplo, teremos "div"s num container de classe jorge, e vamos ser capazes modifica-los mesmo tais divs não havendo identificadores, classes, ids, etc;

    .jorge{ border-color: black; border-style: solid; border-width: 0.1em; }
    .jorge:hover{ background-color: aqua; }
    .jorge > div:nth-child(2){ background-color:darkgoldenrod; }
    .jorge > div:nth-child(3){ background-color:violet; }
    .jorge:nth-child(1){ background-color: blue; }
    Perceba que nth-clild não funciona por não atribuir o contexto corretamente. Mais detalhes olhe o código fonte .css;

    Seletor: Pseudo-Element

    Pseudo-Element tem como objetivo modificar algo dentro do escopo determinado pelo desenvolvedor.
    Pseudo-Elementos são orientados a serem mais específicos, a estilização precisa de certos elementos.
    A lista completa pode ser encontrada aqui: Pseudo-Elements por W3shools

    .jorge{ border-color: black; border-style: solid; border-width: 0.1em; }
    .jorge:hover{ background-color: aqua; }
    .jorge > div:nth-child(2){ background-color:darkgoldenrod; }
    .jorge > div:nth-child(3){ background-color:violet; }
    .jorge:nth-child(1){ background-color: blue; }
