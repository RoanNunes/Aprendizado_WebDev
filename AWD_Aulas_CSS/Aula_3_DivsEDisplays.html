Div(s) e Displays

Div(s) são apenas um tipo de tag de HTML que define uma divisão/container genérico, semelhante conceitualemnte a um article e um section.
No entanto, div(s) são tag genéricas, flexíveis, containers o qual o desenvolvedor define o que ela será, fará e se comportará.
Dito isso, o maior desafio ao manipular Div é como elas se comportam na página e sua relação com outros elementos, incluindo como ela são exibidas ou "displayed".
Sendo assim, temos a propriedade CSS denominada "Display", destacando seguintes valores que podem ser atribuidos:

  • display: block
  • display: inline
  • display: inline-block
  • display: none
  • display: flex
  • display: grid

    • Há outros métodos de display, os quais podem encontrar seguinte link: Lista de valores a Display por W3S
    Cada elemento HTML possui um método específico de display, o qual pode ser modificado. Div por padrão assume o "display: block".
    Para deixar claro nos exemplo, o "body" está estilizado para assumir 75% do comprimento da página (a distância horizontal) e margens para centralizar.
    Toda Div caso não tratada especificamente, irá percorrer todo o "body";
    Devido a sua complexidade, métodos de se comportar, interações com outros elementos e a presença em todo projeto que irá ocorrer, cada display será explicado separadamente.

    Display: block

    "Display: block" é a configuração padrão de todo elemento Div.
    Por definição o valor "block" exibe um elemento como um elemento de bloco (como (p)). Começa em uma nova linha e ocupa toda a largura
    Ao atribuir display: block, ele atribui ao container um width e height distintos os quais podem ser modificados.

    .container-Block{ display: block; background-color: darksalmon; height:200px; }
    Eu sou um display: block;
    Eu sou um display: block;
    Eu sou um display: block;

    Display: inline

    Por definição o valor "inline" exibe um elemento como um elemento embutido (como (span)). Quaisquer propriedades de altura e largura não terão efeito.
    Ao atribuir display: inline, ele não atribui ao container um width e height, e não podem ser modificados.
    Inline assumi o espaço necessário para conter seu conteúdo.

    .container-Inline{ display: inline; background-color: aquamarine; margin: 10px; width: 200px; height:200px; }
    Eu sou um display: inline;
    Eu sou um display: inline;
    Eu sou um display: inline;

    Display: inline-block

    Por definição o valor "inline-block" exibe um elemento como um contêiner de bloco de nível embutido. O próprio elemento é formatado como um elemento embutido, mas você pode aplicar valores de altura e largura.
    Ao atribuir display: inline-block, ele atribui ao container um width e height, no entanto, seu comportamento é como um de elemento de "display: inline"
    Inline assumi o espaço necessário para conter seu conteúdo.

    .container-Inline-Block{ display: inline-block; background-color: rgb(102, 99, 255); margin: 10px; width: 200px; height:200px; }
    Eu sou um display: inline-block;
    Eu sou um display: inline-block;
    Eu sou um display: inline-block;

    Display: none

    Por definição o valor "none" remove a exibição do elemento por completo.
    Ao atribuir display: none o próprio elemento e seus adjacentes se comportam como ele não existe.
    Existe uma diferença entre "display: none" e "visibility: hidden". Um o remove da exibição e seu comportamento, o outro remove apenas sua visibilidade mantendo o comportamento de exibição.

    .container-Inline-Block{ display: inline-block; background-color: rgb(102, 99, 255); margin: 10px; width: 200px; height:200px; } .container-None{ display: none; background-color: rgb(0, 255, 76); margin: 10px; width: 200px; height:200px; }
    Eu sou um display: inline-block;
    Eu sou um display: inline-block;
    Eu sou um display: inline-block;

    Display: flex

    Por definição o valor "flex" é um container responsivo que permite elementos dentro dele crescer ou minimizar.
    Ao atribuir display: flex o container assumi inicialmente propriedades semelhantes ao block.
    A diferença entre os outros métodos é sua adaptabilidade ao espaço delimitado e tamanho da tela exibida.
    Em um "display: flex" ou "flex container" ele possui as seguintes propriedades específicas:

  • flex-direction - A propriedade flex-direction define em qual direção o contêiner deseja empilhar os itens flexíveis.
  • flex-wrap - A propriedade flex-wrap especifica se os itens flexíveis devem ser encapsulados ou não.
  • flex-flow - A propriedade flex-flow é uma propriedade abreviada para definir as propriedades flex-direction e flex-wrap.
  • justify-content - A propriedade justify-content é usada para alinhar horizontalmente os itens flexíveis
  • align-items - A propriedade align-items é usada para alinhar verticalmente os itens flexíveis
  • align-content - A propriedade align-content é usada para reposicionar os elementos dentro do container flex por um método diferente.

    • Pessoalmente recomendo o seguinte:
  • Ao utilizar um container flex (classe ou id), crie classes distintas ou modificadoras que serão os elementos dentro dele para melhor utilização do display: flex.
  • Ao utilizar um container flex, teste o site em múltiplas resolução para verificar o fluxo do site e as interações que ele tem com outros elementos.

    • .container-Flex-Factory-New{ display: flex; background-color: rgb(243, 255, 75); margin: 10px; }
    .container-Flex-Mod{ display: flex; background-color: rgb(255, 75, 225); margin: 10px; width: 500px; height: 500px; }
    .cflex-Perfect-Center{ height: 300px; justify-content: center; align-items: center; }
    .cflex-Elemento-Prenchedor{ align-content: stretch; }
    .cflex-Elemento-Prenchedor > div{ height: auto; }

    Display: grid

    Por definição o valor "grid" oferece um sistema de layout baseado em grade, com linhas e colunas, facilitando o design de páginas da Web sem a necessidade de flutuações e posicionamento.
    Assim como "display: flex", um container grid tem propriedades específicas e tem controle sobre como os elementos inseridos nele.
    Este tópico receberá sua aula dedicada, pois este método de display não é algo "plug and play". Requer um planejamento pensado nele.
    O objetivo dele é dar um layout estruturado e permitir a responsividade dele.
    Segue um breve exemplo do que seria ele:

    .grid-container { display: grid; grid-template-columns: 80px 200px auto 30px; gap: 10px; background-color: #2196F3; padding: 10px; }
    .grid-container > div { background-color: rgba(255, 255, 255, 0.8); text-align: center; padding: 20px 0; font-size: 30px; }
