Responsividade e unidades

No século XXI, a tecnologia evolui (e até 2023 ainda evolui) rapidamente, com ela surgiram diferentes métodos de usar e onde usar máquinas computacionais.
Isso quer dizer que diferentes aparelhos com diferentes especificações tentam acessar e realizar funções semelhantes.
Para Web Design específicamente, isso quer dizer que temos diferentes tamanhos de tela para considerar, e como usuário interage com a "página da web"/"aplicação web".
Com o tempo, diferentes frameworks surgiram para suprir essa nova necessidade, a de ser responsivo a expectativa do usuário.
No entanto com a criação de HTML5 e CSS3, muito dessas capacidades se tornaram nativas das linguagens. Um exemplo claro disso são os displays 'flex' e 'grid'.
Acresentando a isso, temos agora dezenas de unidades para definir o espaço de um elemento em Web Development.
Vamos com calma, um tópico de cada vez.

Unidades de tamanho

Essas unidades são referêntes aos valores que podem ser atribuidos a muitas propriedades principalmente ao .CSS.
Temos dois tipos:

  • Unidades Absolutas - São unidades definitivas e fixas, as quais não alteram por nada e apresentaram a exata medida desejada.
  • Unidades Relativas - São unidades as quais possuem algum ponto de referência para se escalarem e modificarem dinamicamente.

    • Unidades Absolutas

    • cm - centímetro
    • mm - milímetro
    • px - pixel
    • in - inches (1in = 96px = 2.54cm)
    • pt - point (1pt = 1/72 of 1in)
    • pc - picas (1pc = 12 pt)

    Unidades Relativas

    • % - Usa como referência o container o qual o elemento está inserido
    • em - Usa como referência o tamanho da fonte (font-size) do elemento
    • rem - Usa como referência o tamanho da fonte (font-size) do ':root'
    • ex - Usa como referência a altura da fonte (x-height) do elemento
    • ch - Usa como referência o espaçamento do caractere "0" (zero)
    • vw - Usa como referência o comprimento (width) do viewport disponível
    • vh - Usa como referência a altura (width) do viewport disponível
    • vmin - Usa como referência o menor ou mínimo espaço do viewport disponível
    • vmax - Usa como referência o maior ou máximo espaço do viewport disponível

    Responsividade e Adaptabilidade

    Os tópicos responsividade, adaptabilidade, acessabilidade, embora haver teorias, artigos e estudos por academias e estudiosos, alguns consideram eles como temas subjetivos.
    Pessoalmente eu diria o tópico não é subjetivo, mas existe execuções distintas as quais ainda são estudas o métodos mais efetivos.
    Sobre o conhecimento teoríco, recomandaria ler:

  • O livro "The design of everyday things" por Don Norman. É uma boa leitura :)
  • O tema "Design thinking", existe diferentes livros e artigos. Este é um bom tema de entrada para entender certas nuancias da experiência do usuário.

    • Aqui vamos falar mais sobre ferramentas que auxiliam a criação de designes responsivos.

    Viewport

    A viewport é a área visível do usuário de uma página da web.
    A janela de visualização varia de acordo com o dispositivo e será menor em um telefone celular do que em uma tela de computador.
    Antes dos tablets e celulares, as páginas da web eram projetadas apenas para telas de computador, e era comum que as páginas da web tivessem um design estático e um tamanho fixo.
    Então, quando começamos a navegar na Internet usando tablets e telefones celulares, as páginas da Web de tamanho fixo eram muito grandes para caber na janela de visualização. Para corrigir isso, os navegadores nesses dispositivos reduziram toda a página da Web para caber na tela.
    Isso não foi perfeito! Mas uma solução rápida.
    (<)meta name="viewport" content="width=device-width, initial-scale=1.0"(>)
    Segue alguns conselhos por W3Shools sobre isso:

    1. NÃO use elementos grandes de largura fixa - Por exemplo, se uma imagem for exibida em uma largura maior que a janela de exibição, ela poderá rolar horizontalmente. Lembre-se de ajustar esse conteúdo para caber na largura da janela de visualização.
    2. NÃO permita que o conteúdo dependa de uma largura específica da janela de visualização para renderizar bem- Como as dimensões da tela e a largura em pixels CSS variam muito entre os dispositivos, o conteúdo não deve depender de uma largura específica da janela de visualização para renderizar bem.
    3. Use consultas de mídia CSS para aplicar estilos diferentes para telas pequenas e grandes - Definir grandes larguras absolutas de CSS para elementos de página fará com que o elemento seja muito largo para a janela de visualização em um dispositivo menor. Em vez disso, considere usar valores relativos de largura, como largura: 100%. Além disso, tenha cuidado ao usar grandes valores de posicionamento absoluto. Isso pode fazer com que o elemento fique fora da viewport em dispositivos pequenos.

    Media Query

    "Media Query" não é uma classe ou id ou tag. São condições checadas da página Web, caso verdadeiras são realizadas alterações ao .CSS, podendo ser tags, classes, ids, psedo-classes, pseduo-elements e suas propriedades.
    Sintaxe de @media:
    @media not|only 'mediatype' and ('expressions') { CSS-Code; }

    Também é possível determinar uma metadata do @media e direcionar para um arquivo .css dedicado:
    No (Head):
    (link rel="stylesheet" media="mediatype and|not|only (expressions)" href="print.css")

    É possível definir algumas condições, modelo mais usado seria como esses:
    /* Telas muito pequenas (phones, 600px and down) */ @media only screen and (max-width: 600px) {...}
    /* Telas pequenas (portrait tablets and large phones, 600px and up) */ @media only screen and (min-width: 600px) {...}
    /* Telas médias (landscape tablets, 768px and up) */ @media only screen and (min-width: 768px) {...}
    /* Telas grandes (laptops/desktops, 992px and up) */ @media only screen and (min-width: 992px) {...}
    /* Telas extra grandes (large laptops and desktops, 1200px and up) */ @media only screen and (min-width: 1200px) {...}

    Exemplo de uma grid com media query (de 992px)

    Ao modificar o tamanho de exibição da página, nesse contexto, o display grid irá se modificar de 5 colunas para 2 colunas.
    Ou seja, além da adaptabilidade provida por display grid, foi acrescentado mais uma camada de adaptabilidade a esse elemento.
    Isso é só 1 (um) exemplo, existe diversas utilidades para isso, seja criativo!

    Metodologia e Framework

    Existe e serão criadas centenas de metodologia e frameworks para Web Design.
    Isso é algo inevitável pois os padrão do uso e as expectativas do usuário estão em constante evoluação e modificação, tanto para bem quanto para ruim.
    Mesmo nessa condições, é necessário avaliar os métodos mais interessantes para o projeto ou ideia que se deseja ser executado.
    Bootstrap é um framework o qual é muito usado pelo quanto de elementos complexos já criados, como se fosse umua biblioteca.
    No entanto, nem toda ferramenta presente talvez seja de sua necessidade ou não cumpre todos os requerimentos para necessidade do projeto.
    Por isso, na minha opinião, aprenda a base de tudo, que nesse caso é HTML, CSS e JAVASCRIPT (nesse caso pode ser TYPESCRIPT mas isso é outra história) para então ver as ferramentas que a comunidade criou com o intuito de ajudar.
