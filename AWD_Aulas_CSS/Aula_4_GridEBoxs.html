Grid e Boxs

"Display: grid" é uma propriedade a qual requer planejamento antes de seu uso.
Esta propriedade disponibiliza propriedade específicas tanto para o próprio container, quanto os elementos dentro dele, podendo alterar seu comportamento, relação com outros elementos, posicionamento na hieraquia.
Ao ver o código fonte desta página, irá notar a diferença de como é estruturado o HTML em comparação aos modelos anteriores.
Segue a lista de propriedade específicas do container grid:

  • column-gap - Especifica o intervalo entre as colunas
  • gap - Uma propriedade abreviada para as propriedades de intervalo entre linhas e intervalos entre colunas
  • grid - Uma propriedade abreviada para linhas de modelo de grade, colunas de modelo de grade, áreas de modelo de grade, linhas automáticas de grade, colunas automáticas de grade e propriedades de fluxo automático de grade
  • grid-area - Especifica um nome para o item da grade ou esta propriedade é uma propriedade abreviada para as propriedades grid-row-start, grid-column-start, grid-row-end e grid-column-end
  • grid-auto-columns - Especifica um tamanho de coluna padrão
  • grid-auto-flow - Especifica como os itens colocados automaticamente são inseridos na grade
  • grid-auto-rows - Especifica um tamanho de linha padrão
  • grid-column - Uma propriedade abreviada para as propriedades grid-column-start e grid-column-end
  • grid-column-end - Especifica onde terminar o item da grade
  • grid-column-gap - Especifica o tamanho do espaço entre as colunas
  • grid-column-start - Especifica onde iniciar o item da grade
  • grid-gap - Uma propriedade abreviada para as propriedades grid-row-gap e grid-column-gap
  • grid-row - Uma propriedade abreviada para as propriedades grid-row-start e grid-row-end
  • grid-row-end - Especifica onde terminar o item da grade
  • grid-row-gap - Especifica o tamanho do espaço entre as linhas
  • grid-row-start - Especifica onde iniciar o item da grade
  • grid-template - Uma propriedade abreviada para as propriedades grid-template-rows, grid-template-columns e grid-areas
  • grid-template-areas - Especifica como exibir colunas e linhas, usando itens de grade nomeados
  • grid-template-columns - Especifica o tamanho das colunas e quantas colunas em um layout de grade
  • grid-template-rows - Especifica o tamanho das linhas em um layout de grade
  • row-gap - espaço entre linhas

1 -

Entenda que dentro dos elementos, de forma padrão, eles iram se comportar semelhante ao inline-block.
Os elementos dentro do Container GRID irão se modificando conforme seu conteúdo.

2 -

É possível modificar tal conteudo, através da determinação de valores absolutos ao elemento.
Por isso vamos o que compõe um elemento em questão de espaço.
Temos quatro opções para modificar o espaço ocupado de um elemento:

3 -
  1. Margin (ou margem)
  2. Border (ou borda)
  3. Padding (ou prenchimento)
  4. Content (ou conteudo)
4 -

"margin: value" - Esta define o quanto de margem presente na caixa. 'Margin' é o espaçamento (ou preenchimento) externo a ao container/elemento.

5 -

"border: value" - Esta define o quanto de borda ou contorno presente na caixa. 'Border' é o preenchimento o qual afeta o espaço tanto externo quanto interno do container/elemento.

6 -

A propriedade "border" possui suas próprias propriedades de estilização.

7 -

"padding: value" Esta define o quanto de preenchimento presente na caixa. 'Padding' é o espaçamento (ou preenchimento) interno a ao container/elemento.

8 -

"Content" se refere aos outros elementos presentes no container, cada elemento dentro do container consome ou acrescenta espaço na área de conteúdo.

9 -

Para sua ciência, existe uma propriedade chamado "content: type". Esta altera outros elementos presentes na class/id, não estando relacionado diretamente a manipulação do espaço.

    Propriedades a ELEMENTOS de uma grid

    Elementos pertencentes ao um container com "display: grid" ganham acesso a propriedades .css específicas as quais iteragem com o container e relacionamento com elementos adjacentes.
    Estas são propriedades presentes no elemento dentro de um container grid, no entanto, modificam como container grid trata ele em relação a outros elementos.
    Lembrando que por ser propriedades específicas ao 'display: grid', elas possuem nomeclaturas específicas.
    Segue as propriedades destacadas:

    • "grid-column: value / value" - Esta propriedade define o quanto um elemento é espaçado em referência a colunas, definindo onde começa e onde termina.
      • [ANOTAÇÃO] 'grid-column: value / value;' é uma propriedade que une as funções das propriedades "grid-column-start: value" e "grid-column-start: value";
    • "grid-column-start: value" - Esta propriedade define o começo do espaçamento de um elemento em referência a colunas.
    • "grid-column-end: value" - Esta propriedade define o fim do espaçamento de um elemento em referência a colunas.
    • "grid-row: value / value" - Esta propriedade define o quanto um elemento é espaçado em referência a linhas, definindo onde começa e onde termina.
      • [ANOTAÇÃO] 'grid-row: value / value;' é uma propriedade que une as funções das propriedades "grid-row-start: value" e "grid-row-start: value";
    • "grid-row-start: value" - Esta propriedade define o começo do espaçamento de um elemento em referência a linhas.
    • "grid-row-end: value" - Esta propriedade define o fim do espaçamento de um elemento em referência a linhas.
    • "grid-area: value / value / value / value" - Esta propriedade define o quanto um elemento é espaçado em referência a linhas e colunas respectivamente.
      • [ANOTAÇÃO] "grid-area: value (linha) / value (coluna) / value (linha) / value (coluna);" é uma propriedade que une as funções das propriedades "grid-row: value / value" e "grid-column: value / value";
      [ANOTAÇÃO] "grid-area: 'string';" é uma alternativa de descrever a área de um elemento, considerando que o continer tenha usasdo como referência "grid-template-areas"
      [ANOTAÇÃO] "grid-area: 'string';" e "grid-template-area": 'string' ... ; são complexos, são outra metodologia de determinas espaços, no entanto, bastante poderoso.

    .main-Grid-Columns{ display: grid; grid-template-columns: 40% 40% 20%; }
    .main-Grid-Columns-Mod-3{ grid-template-columns: 10% auto auto auto auto; grid-gap: 15px 15px; }
    .grid-Element-1{ grid-column: 1 / 4; }
    .grid-Element-2{ grid-row: 2 / 5; }
    .grid-Element-3{ grid-column: 4 / 6; grid-row: 2 / 5; }
    .grid-Element-4{ grid-area: 6 / 3 / 12 / 5; }

    grid-template-areas, complexo, poderoso

    Tanto a propriedade de 'grid-template-areas' e 'grid-area' são muito poderosas permitem uma flexibilidade de manipulação de elementos não vista em comparação aos outros displays.
    Cada propriedade possui métodos específicos, nomeclaturas específicas e manipulam elementos dentro de um container grid de forma bem livre.
    Por serem propriedades interligadas em função, será dificil exemplificar cada propriedade individualmente. Aqui será feito possível para explicar de forma didática.
    Sobre 'grid-template-areas':

    • 'grid-template-areas: ' 'string' ';' - Esta propriedade é definida por "strings", por nomes, por texto, não valores númericos
      • [ANOTAÇÃO] 'grid-template-areas' definido somente no container grid;
      [ANOTAÇÃO] É possivel em 'grid-template-areas' determinar o número de linhas e o número de colunas de um container grid, mas não o espaçamento deles.
    • Para determinar valores a 'grid-template-areas', definimos linhas através de (') aspas simples, e pulamos a linha através do espaço (barra de espaço);
      • [EXEMPLO] grid-template-areas: 'header header logo header header' ;
      [EXEMPLO] grid-template-areas: 'header header logo header header' 'cnt-principal cnt-principal cnt-principal cnt-principal propaganda' 'footer footer footer footer footer';
    • Os titulos/nomes dados a grid-template-areas, tem termos de função, são diferentes de classes e id.
    • [RECOMENDAÇÃO] Os nomes das areas serem semelhantes aos nomes das classes para melhor identificao
    • Pela dinamicidade provida pelo container grid e 'grid-template-areas', torna-se possível a alteração da ordem visual dos elementos dentro dele alterando somente os nomes pelo 'grid-template-areas'
      • [ANOTAÇÃO] Isso é maior benefício desta propriedade, a possibilidade de modificar com precisão usando apenas uma propriedade e mudar radicalmente de forma desejada.
    • A determinação de áreas em 'grid-template-areas' precisa ter um inicio determinado e um final determinado, sem repetição das áreas em meio ao template e sem quebra dá area por outras áreas.
      • [EXEMPLO] Isso funciona (exemplo de linha): 'header header logo . .' (5 colunas e 1 linha: 2 colunas para header; 1 coluna para logo; 2 colunas neutras );
      [EXEMPLO] Isso não funciona (exemplo de linha): 'header header logo header header' (5 colunas e 1 linha: 2 colunas para header; 1 coluna para logo; 2 colunas para header);
    • Uma 'grid-area' é unica no 'grid-template-areas'.
    • Caso queira determinar um coluna em 'grid-template-areas' sem determinar uma área, utilize (.) como se fosse uma área.
    • Caso haja um elemento dentro do container grid de 'grid-template-areas', o qual não foi especificado uma área ele irá povoar as áreas indeterminadas primeiro, depois adicionar elementos ao grid seguindo a contagem de colunas.
    .main-Grid-Areas{ display: grid; grid-template-areas: 'header header logo . .' 'cnt-principal cnt-principal cnt-principal cnt-principal propaganda' 'footer footer footer footer propaganda'; grid-gap: 15px 15px; background-color: aqua; }
    .grid-item-Header{ grid-area: header; background-color: darkgoldenrod; }
    .grid-item-Logo{ grid-area: logo; background-color: rgb(184, 11, 175); }
    .grid-item-Conteudo-Principal{ grid-area: cnt-principal; background-color: rgb(184, 11, 11); }
    .grid-item-Container-Propaganda{ grid-area: propaganda; background-color: rgb(86, 11, 184); }
    .grid-item-Footer{ grid-area: footer; background-color: rgb(60, 184, 11); }

    Considerações finais sobre Grid

    Olha só o .CSS já um assunto vasto, e 'display: grid' por si só possue inúmeras utilizações, interações e modelos.
    Eu penso que com o conhecimento desta aula seja possível ter uma base sólida de conhecimento para fazer experimentos e descobrir suas peculariedades.
    Por exemplo, é possível usar containers grid dentro de containers grid; Modificar a estrutura e ordem dos elementos de acordo com tamanho da tela; acresentar elementos dinâmicamente e etc;
    Por isso, recomendo nessa aula principalmente ver o código fonte (.html e .css) e modificar, estudar, acrescentar ser sua base de experimentos.
