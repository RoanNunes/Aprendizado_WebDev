Considerações inicias

Vou ser sincero, acho que até aqui, você já é capaz de produzir projetos em HTML e CSS.
Caso não saiba de como fazer algo, ou usar certa propriedade ou talvez já existe um método testado, você agora possui uma base de conhecimento prático sólido.
A partir daqui são conhecimentos que eu pessoalmente acho interessante possuir e saber como usar.

Variáveis!

O próprio .CSS possui uma versão de variavéis, assim como as linguagens de programação, embora sejam mais limitadas.
Toda variável deve ser declarada no ':root' e definida um valor a ela.
Sua nomeclatura é distinguida com o prefíxo (--) dois traços.
Pode ser utilizar várias vezes em meio ao CSS e para sua utilização, usamos 'var('--nomeVariavel')'

Exemplo:

:root{ --souVariavel: 12px; }
.classeDeExemplo{ width: var(--souVariavel); height: var(--souVariavel); }
#IdDeExemplo{ font-size: var(--souVariavel); margin: var(--souVariavel); }
h7{ padding: var(--souVariavel) var(--souVariavel); }

Toda variável é necessário que tenha um valor definido a ela.
Este valor pode ser todo ou qualquer valor aceito por alguma propriedade .CSS, no entanto, nem toda variável pode funcionar em qualquer propriedade por causa desse valor.

As variáveis CSS podem ter um escopo global ou local!
As variáveis globais podem ser acessadas/utilizadas em todo o documento, enquanto as variáveis locais podem ser utilizadas apenas dentro do seletor onde é declarada.
Para criar uma variável com escopo global, declare-a dentro do seletor :root. O seletor :root corresponde ao elemento raiz do documento.
Para criar uma variável com escopo local, declare-a dentro do seletor que vai utilizá-la.

Variáveis também podem ser manipuladas pelo Javascript, dependendo do espoco apropriado a ela!

Animação!

O CSS permite que a criação de animações, movimentação e modificação de um elemento sem a necessidade de Javascript ou outro Framework!
No passado, essa área era predominada por arquivos do Adobe Flash, mas ele foi descontinuado em 2020. Meus pesames.
A saída dele ocorreu devido a evolução do próprio HTML e CSS, e por terem surgido frameworks com mais capacidade e flexibilidade do que ele, além do surgimento da plataforma mobile.

Aqui não vamos falar de frameworks, vamos falar de CSS3 puro.

Em primeiro, temos a regra '@keyframe', assim como média, mas com suas próprias distinções e utilizações.

  • @keyframes
    • É a regra, onde é especificado o código de animação
  • animation
    • Uma propriedade a qual é definida no elemento o qual ocorrerá animação. Esta abreviada para definir todas as propriedades de animação
  • animation-delay
    • É uma propriedade a qual especifica um atraso para o início de uma animação
  • animation-direction
    • É uma propriedade a qual especifica se uma animação deve ser reproduzida para frente, para trás ou em ciclos alternados
  • animation-duration
    • É uma propriedade a qual especifica quanto tempo uma animação deve levar para completar um ciclo
  • animation-fill-mode
    • É uma propriedade a qual especifica um estilo para o elemento quando a animação não está sendo reproduzida (antes de começar, depois de terminar ou ambos)
  • animation-iteration-count
    • É uma propriedade a qual especifica o número de vezes que uma animação deve ser reproduzida
  • animation-name
    • É uma propriedade a qual especifica o nome da animação @keyframes a qual vai ser utilizado no elemento.
  • animation-play-state
    • É uma propriedade a qual especifica se a animação está em execução ou pausada
  • animation-timing-function
    • É uma propriedade a qual especifica a curva de velocidade da animação

@keyframe e Classes exemplos

Exemplo 1

@keyframes simple-LefttoRight {
from{ left: 0%; }
to{ left: 84%; }
}
.box-Active-1{ position: relative; background-color: pink; text-align: center; width:15%; padding: 15px 0px; margin: 20px 0px; animation-name: simple-LefttoRight; animation-duration: 5s; }

Exemplo 2

@keyframes simple-GrowEColor {
0%{ left: 0%; width: 15%; background-color: chartreuse; }
50%{ left: 50%; width: 50%; background-color:darksalmon; }
90%{ left: 0%; width: 10%; background-color:darkgreen; }
100%{ left: 0%; width: 15%; background-color:chartreuse; }
}

.box-Active-2{ position: relative; background-color: rgb(156, 199, 0); text-align: center; width: 15%; padding: 15px 0px; margin: 20px 0px; animation-name: simple-GrowEColor; animation-duration: 5s; animation-iteration-count: infinite; }

Algumas anotações:

  • Perceba que há mais de um método de descrever o "fluxo do tempo" de uma animação, seja "from/to" ou "0%/100%".
  • Se for uma propriedade CSS, ela provavelmente é uma propriedade 'animável'.
  • É possível atribuir uma animação dado um estado específico do elemento, com psudo-classes e psudo-element.
  • Atenção com a sobreposição de elementos, e a ordem dos elementos

    • Exemplo 3

    @keyframes MoveERotate {
    0%{ left: 0%; rotate: 0deg; }
    10%{ left: -4%; rotate: -10deg; }
    30%{ left: 80%; rotate: 405deg; }
    50%{ left: 80%; rotate: 410deg; }
    100%{ left: 0%; rotate: 0deg; }
    }
    .box-Active-3{ position: relative; background-color: rgb(0, 199, 172); text-align: center; width: 15%; padding: 15px 0px; margin: 20px 0px; }
    .box-Active-4{ position: relative; background-color: rgb(143, 0, 199); text-align: center; width: 20%; padding: 25px 0px; margin: 50px 0px; }
    .box-Animation-MoveERotate{ animation-name: MoveERotate; animation-duration: 4s; animation-iteration-count: infinite; }
